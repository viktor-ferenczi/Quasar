# Quasar.Agent/GridRenderSceneInspector.cs

**Module:** Quasar.Agent  **Kind:** class  **Tier:** 1

## Summary
Game-thread helper that builds viewer scene snapshots from live `MyCubeGrid` and asteroid `MyVoxelBase` instances. It maps primary and optional context grids into `Magnetar.Protocol` viewer DTOs, computes selected-grid-local context bounds plus matching relative and enclosing world AABBs, enumerates loaded intersecting grids deterministically, clips surrounding grids at block selection time with oriented context bounds and conservative context caps, registers active scene mod asset roots from `MySession.Static.Mods`, registers logical model and texture names with mod root hints from definition contexts or normalized `ModsPath` paths, includes generated cube-part and runtime subpart metadata, captures selected-grid conveyor/logistics nodes, physical conveyor links, small-line restrictions, working state, roles, and connected systems, captures terminal-synced lighting metadata from classic `MyLightingBlock`, component-based `MyLightingComponent`, and runtime subpart lighting-logic sources using server working state instead of client render fade power, captures LCD metadata from standalone `MyTextPanel` and `IMyMultiTextPanelComponentOwner` blocks, and assigns grid ownership on blocks, chunks, and lights. When requested for grids, it samples bounded voxel content/material ranges intersecting the floor-grid footprint or the enclosing context world AABB, chooses a coarser storage LOD when needed to fit scene chunk/byte limits, and preserves surfaces that cross internal chunk boundaries where possible so the browser can procedurally mesh them. For standalone asteroid voxel maps, it emits a synthetic grid anchor, chooses the lowest storage LOD that fits separate body chunk/byte limits, samples the selected voxel storage in chunks, skips uniform empty/full chunks, and resolves stale/direct `MyVoxelPhysics` requests through their root voxel before rejecting planets. It intentionally avoids client-render-only dependencies such as `VRage.Render` texture-change payloads or offscreen LCD texture bytes and does not expose raw game asset files, inventory contents, or server-local mod paths.
